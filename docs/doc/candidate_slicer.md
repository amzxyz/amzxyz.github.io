# ⚔️ 候选切割机 (Lua)：输入过程中的“精准手术刀”

> **痛点场景：** 当你行云流水地打出一长串拼音，却发现因为发音相似或模型数据偏差，长句的后半截“偏航”了。按退格键全部删掉重打？太低效！此时，你需要这把“候选切割机”。

万象通过底层 Lua 脚本，实现了极具确定性的“半截上屏”操作。截断正确的部分，让剩余的编码自我救赎。

---

## 🎯 一、 核心机制与操作指南

当候选框出现长句，且**首选词的前 N 个字完全正确**，但后续连带的词出现错误时，请停止敲击空格，直接使用组合键切割：

* **操作方式**：按下 **`Ctrl + 1~0`**（数字键 1 到 0 代表 1 到 10 个字）。

* **执行效果**：

  1. 系统会立刻将首选项的**前 N 个字直接上屏**。

  2. 剩下的拼音编码会按照音节自动分割，并保留在输入区。
  
  3. **奇妙反应**：由于前面产生干扰的编码已被剥离上屏，残留下来的编码通常会自动“拨乱反正”，瞬间变成你想要的正确选项。此时你可以直接敲空格上屏，或者按 `Tab` 键继续补充辅助码。

**💡 极速实战演示：**
> 你输入了一长串拼音，首选出现了 `今天下午开毁`（本来想打“今天下午开会”）。
> 此时直接按下 **`Ctrl + 4`**：
> 👉 `今天下午` 立刻上屏！
> 👉 输入框残留 `kai'hui`，由于失去了前面的语境干扰，首选自动变成了正确的 `开会`，按下空格，完美收工！

## ⚖️ 二、 版本差异与进阶对策 (Base vs Pro)

切割机在不同的版本中，表现出的“脾气”略有不同，请根据你所使用的版本调整预期：

* 🟢 **标准版 (Base)**：因为只涉及纯粹的双拼/全拼，候选组合具有**极高的确定性**。在长句连打中，切割机简直是行云流水的神器，盲切成功率极高。

* 🔵 **增强版 (Pro)**：由于 Pro 版加入了强力的“辅助码系统”，辅助码的介入会强制聚拢特定的候选词。此时如果强行切割，残留的编码可能会因为受到辅助码的束缚，得出与你预期不符的候选。

  * **Pro 版自救指南**：如果切割后没得到想要的结果，你可以尝试多次按下 **单引号 `'`** 来手动循环分词；但更高效、更具极客直觉的做法是——**直接按下 `Esc` 清除残留，重新输入后半段**。
