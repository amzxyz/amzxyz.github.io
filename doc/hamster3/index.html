    📱 Hamster3 (元书输入法) 部署指南

    欢迎在 iOS 平台使用万象。元书输入法 (Hamster3) 是目前万象官方最为推荐的 iOS 端前端软件，其性能表现与方案管理逻辑在移动端出类拔萃。

    1. 启用键盘与完全访问权限

    1. 安装元书输入法后，前往系统 设置 → 通用 → 键盘 → 键盘 → 添加新键盘，添加 Hamster。
    2. 点击 Hamster 键盘项，务必开启“允许完全访问”。
      • (⚠️ 注意：若不开启，万象 Pro 版本的双拼切换及 Lua 魔法插件将无法正常运行。)

    2. 建立专属万象目录

    元书支持多套方案共存。为了保持文件整洁，我们需要建立独立目录。

    1. 打开 iOS 自带的 文件.app。
    2. 进入路径：我的iPhone/元书输入法/RimeUserData。
    3. 在此目录下新建文件夹，推荐命名为：wanxiang。

    3. 导入万象方案

    打开元书输入法 APP，进入 【输入方案】 页面：

    • 方式 A (推荐脚本/链接)：点击右上角菜单按钮，选择 【下载方案】，填入万象 Release 中对应的方案 .zip 包下载链接。
    • 方式 B (手动导入)：提前下载好 zip 包后，选择 【导入方案】 并选取该文件。

    关键操作：导入完成后，再次点击右上角菜单，选择 【方案目录切换】，选中你刚才创建的 wanxiang 目录。

    4. 执行部署与基础配置

    1. 在【输入方案】页面，点击菜单旁的 “刷新” 按钮完成初次部署。
    2. 皮肤与按键处理请参考仓输入法设置，确保斜杠 / 等符号交由 Rime 处理。

    5. 🚨 核心必读：键盘文件与应用文件的同步博弈

    元书输入法在每次部署前会自动将键盘文件复制到应用目录。但默认仅复制 text 和 userdb（词库数据），不会复制 .yaml 配置文件。

    如果你希望通过斜杠指令修改配置并永久生效，请根据你的使用习惯选择以下方案：

    适合人群：希望在键盘上输入指令后，部署即可永久生效，不打算手动编辑代码的用户。

    1. 进入 我的iPhone/元书输入法/RimeSharedSupport。
    2. 复制 include_keyboard_rime_files.txt 到你的 wanxiang 目录下。
    3. 编辑该文件，在底部新增一行： 
      ^.*[.]custom.*$
    4. 效果：后续在键盘使用斜杠指令产生的 custom 修改将自动同步回应用文件。

    局限性警告

    此模式下，应用目录中的 custom 文件在部署时会被键盘文件覆盖。如果你在电脑上写了高阶配置传进手机，可能会被手机旧的键盘记录抹除。

    适合人群：习惯在电脑端或文本编辑器中精细打磨配置文件，且不希望被键盘临时记录干扰的用户。

    • 操作方式：无需修改任何正则文件。
    • 后果：你不能通过键盘上的斜杠指令来快速切换方案。
    • 建议：所有针对 custom 文件的修改，请直接使用 iOS 文件管理或第三方编辑器在 RimeUserData/wanxiang 目录下手动编辑。
    • 注意：custom 目录下文件只是示例，你需要复制到 RimeUserData/wanxiang 目录下手动编辑。

    6. 初始指令与输入类型切换

    (若你选择了路线 A，请执行以下指令；路线 B 用户请手动修改配置文件。)

    万象的默认状态如下：

    • 🟢 Base 标准版：默认开启 全拼。
    • 🔵 Pro 增强版：默认开启 自然码双拼。

    强烈建议执行一次激活指令

    即使默认方案符合需求，也建议执行切换指令以激活底层四个方案文件的关联配置。

    👉 操作方法：直接在键盘中文状态输入指令，然后点击 APP 内的“刷新”部署。
